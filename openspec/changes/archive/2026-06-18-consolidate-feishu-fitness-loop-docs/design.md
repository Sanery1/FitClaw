## Design

Create a new `docs/FEISHU_FITNESS_LOOP_STATUS.md` document as the current handoff for the Feishu-first fitness assistant.

The document should cover:

- product framing;
- current user loop;
- eval coverage map;
- remaining gaps;
- recommended next stage;
- verification commands.

Update `docs/PROJECT_UNDERSTANDING.md` only enough to point future readers to the new status doc and reflect that the Feishu loop has moved beyond the older May baseline.

## Risk

Low. This is a documentation-only tweak. The main risk is overstating runtime readiness, so the document should distinguish deterministic eval coverage from real Feishu/live-model validation.
